# IA para Desenvolvimento + Revisão de Estruturas de Dados I (Desafios de Code Interview)

Período de execução: durante os feriados (16/fev e 09/mar) e semanas seguintes.
[Cronograma](https://trello.com/invite/b/698b3379315d88053e46da1d/ATTI82248eda9527778ae9437cc82aeebab2323C77A7/cronograma-revisao-de-estruturas-de-dados-i)

---

## Pré-requisitos

- Programação básica/intermediária (Python)
- Conhecimentos iniciais de arrays, listas encadeadas e pilhas
- Capacidade de ler e interpretar enunciados de problemas de programação


## Regra central - O uso de IA para desenvolvimento de software é obrigatório

---

## Objetivos

1. Realizar o primeiro contato estruturado com IA como ferramenta de desenvolvimento de software (planejamento, solução, revisão e testes).
2. Revisar Estruturas de Dados I de forma prática por meio de problemas clássicos de entrevistas técnicas.

---

## Parte 1 — Escolha de uma IA para desenvolvimento (precisamos decidir)

Cada equipe deve escolher uma ferramenta de IA voltada para desenvolvimento (uma principal), por exemplo:

- ChatGPT  
- GitHub Copilot  
- Cursor  
- Claude  
- Gemini  
- Codeium  
- Tabnine  
- Continue (plugin open-source)  
- Aider (IA no terminal)

Cada integrante deve produzir um relatório, registrar (com prints legíveis e aptos para adicionar nos documentos e apresentação):

- Nome da ferramenta  
- Como foi acessada (web, IDE, plugin, terminal)  
- Por que foi escolhida para esta atividade  
- Leu algum tutorial? Assistiu algum vídeo? É integrada a sua IDE (VSCode, PyCharm, outra...)  
- Como a equipe pretende usá-la (ex.: gerar rascunho, revisar, sugerir testes, explicar complexidade, identificar erros)

---

## No relatório, incluir para cada problema

- Nome do problema  
- Link do problema  
- Plataforma utilizada  
- Estrutura de dados principal usada  
- Justificativa: por que essa estrutura é adequada para resolver o problema  

---

## Parte 2 — Seleção de desafios de programação (obrigatório - precisamos decidir)

Cada equipe deverá escolher desafios de programação no estilo “code interview”, utilizando as estruturas de dados da revisão.

Os problemas devem ser retirados de plataformas como:

- LeetCode  
- HackerRank  
- CodeSignal  
- CodeWars  
- NeetCode  
- GeeksForGeeks  
- CSES Problem Set  
- AtCoder (nível iniciante)  
- Beecrowd (URI)  
- AlgoExpert (se houver acesso)

---

## Quantidade de problemas

12 problemas no total, distribuídos da seguinte forma:

- 5 problemas de Arrays  
- 4 problemas de Listas Encadeadas  
- 3 problemas de Pilhas  

| Integrante     | Pilhas | Arrays | Listas Encadeadas |
|---------------|--------|--------|------------------|
| Alexandre     | 1      | 2      | 1                |
| Maria Alice   | 1      | 1      | 2                |
| Mariele       | 1      | 2      | 1                |
  

---

## Regras adicionais

- Os problemas devem ser, preferencialmente, de nível fácil ou médio.
- Pelo menos metade dos problemas deve ser de nível médio (para garantir revisão consistente).
- Os problemas não podem ser repetidos dentro da equipe.
- Cada integrante deve ser responsável por pelo menos 3 problemas.
- Todo código no GitHub da equipe, todos membros da equipe devem ter commits realizados.
